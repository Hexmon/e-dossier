# Governance Docs

Audience: maintainers, contributors, and repository owners.

## What Belongs Here

- contribution process and repo workflow rules
- documentation standards
- security process documentation
- ownership and quality expectations for docs

## What Does Not Belong Here

- implementation walkthroughs
- operational deployment runbooks
- runtime Help Center content

## Sections

- `how-to/` for contributor process guides
- `reference/` for standards and policy-style material
